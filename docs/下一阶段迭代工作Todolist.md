# 下一阶段迭代工作 Todolist

更新日期：2026-05-21

## 模板设计器最终形态

目标：从“编辑单页病历样式”升级为“面向医院病历文书的可视化模板平台”，由模板设计工作台、模板运行引擎、模板版本/发布中心、模板预览测试中心共同支撑。

## MAGI 三脑轮次

### 1. 审视

- [x] 梳理现有模板 DSL、设计器、规则引擎、值提取、模板管理能力。
- [x] 确认已具备文书结构、页面版式、字段绑定、运行预览的基础雏形。
- [x] 识别当前最缺平台化支点：模板版本/发布状态与发布前校验。

### 2. 执行

- [x] 为模板注册表补充草稿、待发布、已发布、已撤回状态。
- [x] 为模板注册表补充版本历史记录与已发布模板查询能力。
- [x] 发布/送审前接入 schema 校验，阻断重复字段、悬空规则引用等配置问题。
- [x] 在模板中心卡片展示状态、版本、配置问题数量。
- [x] 在模板中心接入送审、发布、撤回操作。
- [x] 增加发布流程单元测试。

### 3. 提升

- [x] 将“使用模板”运行端默认收敛到已发布模板，草稿只允许在预览测试中心使用。
- [x] 设计模板预览测试中心：测试数据集、规则触发面板、必填校验结果、字段值导出。
- [x] 设计模板版本中心：版本对比、发布备注、回滚、复制为新版本。
- [x] 设计病历规则中心：必填、只读、显隐、级联、签名、时效、权限、质控规则统一编排。
- [x] 设计医院业务字段中心：患者主索引、就诊信息、诊断、医嘱、检验检查、字典项绑定。
