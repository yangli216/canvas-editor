# 模板管理深度优化方案

更新日期：2026-05-21

## 设计定位

模板管理不再是简单的模板弹窗列表，而是医院病历文书模板资产的管理工作台。它需要支持设计、查找、测试、发布、回滚、规则治理和业务字段治理。

## 借鉴对象

- Linear：清晰的侧边导航、低噪声状态表达、主工作区聚焦任务。
- Airtable：数据资产以卡片和列表双视图组织，适合不同密度的浏览方式。
- Ant Design Pro：企业后台常见的统计概览、筛选工具栏和表格/卡片混合管理模式。
- Notion 模板库：模板卡片强调名称、说明、标签和快速动作，降低选择成本。

## 体验问题审视

1. 原模板管理是 780px 小弹窗，空间不足，无法承载版本、发布、规则、字段等平台能力。
2. 分类以横向 Tab 展示，分类增多后不易扫描，也缺少状态维度。
3. 模板卡片信息弱，只展示名称、描述、分类和按钮，无法快速判断发布状态、质量风险、结构复杂度。
4. 操作按钮平铺在卡片底部，主次关系不清晰。
5. 缺少搜索、排序、状态筛选、结果统计和空状态引导。

## 优化目标

1. 视觉：从普通弹窗升级为工作台，采用侧栏 + 主区 + 洞察卡 + 工具栏 + 内容区。
2. 交互：支持分类筛选、流程状态筛选、关键词搜索、排序、卡片/列表视图切换。
3. 信息：在卡片上展示状态、版本、分类、结构块数、字段数、规则数、质量状态和更新时间。
4. 行为：发布模板走“使用”，非发布模板走“测试”，减少误用草稿模板的风险。
5. 扩展：保留版本中心、规则中心、业务字段中心入口，为后续平台化能力留出稳定位置。

## 已落地改造

### 工作台布局

- 模板中心标题升级为“模板管理工作台”。
- 左侧增加分类与流程状态侧栏。
- 主区增加统计洞察：当前结果、已发布、待发布、需修正。
- 内容区支持卡片视图与列表视图。

### 筛选与查找

- 支持按模板名称、描述、分类、版本、状态搜索。
- 支持按草稿、待发布、已发布、已撤回筛选。
- 支持按最近更新、名称、流程状态排序。

### 卡片重构

- 使用状态色条表达生命周期。
- 使用质量标签表达“可发布 / 待修正”。
- 展示结构块、字段、规则三个核心复杂度指标。
- 保留版本、编辑、发布、撤回、删除等操作，同时强化“使用 / 测试”主操作。

### 响应式体验

- 大屏保持侧栏 + 主区工作台。
- 窄屏自动变成上下布局，侧栏高度受控，卡片列表保持可读。

## 验收清单

- [x] 打开模板管理后能看到工作台化布局。
- [x] 分类筛选和流程状态筛选能刷新模板列表与统计。
- [x] 搜索能按名称、描述、分类、版本、状态过滤。
- [x] 排序能切换最近更新、名称、流程状态。
- [x] 卡片/列表视图切换不丢失筛选条件。
- [x] 模板卡片能展示状态、版本、分类、质量、结构块、字段、规则、更新时间。
- [x] 已发布模板执行“使用”，非发布模板执行“测试”。
- [x] 版本中心、规则中心、业务字段中心入口保留并与工作台视觉一致。

## 下一步建议

1. 将模板列表持久化为真实后端分页接口，支持医院、科室、文书类型多维筛选。
2. 增加模板缩略预览图，减少仅凭名称选择模板的认知成本。
3. 增加批量操作：批量发布、批量撤回、批量导出、批量归档。
4. 增加最近使用、收藏、我创建的模板等个人效率视图。
5. 为规则中心和业务字段中心补充真实编辑能力，而不是仅展示治理入口。
