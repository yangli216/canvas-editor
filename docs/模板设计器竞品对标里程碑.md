# 模板设计器竞品对标里程碑

更新日期：2026-05-21

## 1. 调研

本轮对标对象聚焦成熟低代码与专业设计工具的共性能力：

- Webflow / Framer：右侧属性面板强调上下文配置、快速套用样式、即时视觉反馈。
- Retool / Appsmith：组件属性以预设、开关、枚举和帮助提示降低配置门槛。
- Figma：选中对象后快速应用样式 token，减少重复手工调参。
- Ant Design Form Builder / Formily Designer：字段类型、校验、默认值、选项等表单能力需要被高频配置并可复用。

结合医院病历文书场景，下一里程碑不追求泛化组件堆叠，而是优先提升维护人员在右侧属性栏里的配置效率、理解效率和一致性。

## 2. 规划

下一里程碑定义为“专业低代码属性配置台”。目标如下：

- 功能：高频字段配置可一键快配，减少从零选择类型、宽度、占位符、选项的重复操作。
- 交互：文本样式支持预设卡片，维护人员可以先套用正文、标题、提示、风险强调等常见样式，再做微调。
- UI：预设入口以紧凑卡片呈现，保持右侧面板密度，不打断已有行级配置流。
- 可维护性：文档同步记录竞品对标路径，便于后续继续按“调研、规划、执行、验收”推进。

## 3. 执行

已落地能力：

- 字段属性新增“字段快配”区域，提供短文本、长文本、日期、数值、是/否选择、签名等常见业务字段预设。
- 字段快配会保留当前字段 ID 和标签，只套用类型、宽度、占位符、选项、必填等维护成本高的配置。
- 文本样式区域新增样式预设，覆盖正文、一级标题、提示说明、风险强调。
- 样式预设以卡片方式展示，并能高亮当前命中的预设，方便维护人员判断当前配置是否偏离标准。
- 预设卡片做紧凑化处理，适配右侧属性面板宽度。

## 4. 验收

- [x] 选择字段后，右侧属性栏能看到字段快配卡片。
- [x] 点击字段快配后，字段 ID 和标签保持不变，类型、宽度、占位符、选项等配置被快速套用。
- [x] 文本样式配置区域能看到正文、一级标题、提示说明、风险强调等样式预设。
- [x] 当前样式命中预设时，预设卡片会进入选中态。
- [x] 预设能力与默认值占位、说明 tooltip、失焦提交等既有属性面板交互兼容。

## 后续演进

1. 为字段快配接入医院业务字段中心，实现“字段定义 + 数据绑定 + 控件配置”一键套用。
2. 增加块级模板片段收藏，把当前选中块或分节保存为团队可复用片段。
3. 增加属性面板分组折叠和搜索，在大型字段配置中快速定位目标配置项。
4. 将样式预设升级为医院级设计 token，支持科室模板统一风格治理。
